# Data Sources

This project uses public World Cup CSVs from DataHub's `football/worldcup` dataset, which is
sourced from the Fjelstul World Cup Database by Joshua C. Fjelstul, Ph.D. It also uses the
open `martj42/international_results` dataset for Phase 1 current-strength features.

## Raw files

Raw downloads are stored under `data/worldcup/raw/`. These files should remain unchanged so the
feature pipeline is reproducible.

| Local file | Download URL | Purpose |
| --- | --- | --- |
| `data/worldcup/raw/tournaments.csv` | https://datahub.io/football/worldcup/_r/-/tournaments.csv | Tournament metadata such as year, host, winner, and format flags |
| `data/worldcup/raw/teams.csv` | https://datahub.io/football/worldcup/_r/-/teams.csv | Team IDs, names, codes, federations, and confederations |
| `data/worldcup/raw/matches.csv` | https://datahub.io/football/worldcup/_r/-/matches.csv | Match-level results, teams, stages, dates, and scores |
| `data/worldcup/raw/tournament_standings.csv` | https://datahub.io/football/worldcup/_r/-/tournament_standings.csv | Top-four tournament result positions |
| `data/worldcup/raw/qualified_teams.csv` | https://datahub.io/football/worldcup/_r/-/qualified_teams.csv | One row per qualified team per tournament, used for appearances and stage performance |

## Processed outputs

Processed outputs are generated by:

```bash
npm run build:features
```

The command writes cleaned and feature-engineered tables under `data/worldcup/processed/`.

Team identity normalization decisions are documented in `NORMALIZATION_RULES.md`. Raw files remain
unchanged; normalized identities are applied only to processed feature outputs.

## Data notes

- The downloaded dataset includes both men's and women's World Cup tournaments. Processed rows include
  `tournament_gender` so the app can filter to men's 2026 work later.
- `tournament_standings.csv` contains 120 rows, which corresponds to top-four placements across the
  downloaded tournaments. It is not a full final-placement table for every qualified team.
- Because full final placements are not present in the downloaded core tables, the feature pipeline
  exposes `average_finish_position_estimate`, not an undisclosed exact final placement average.
- Penalty shootout goals are not added to goals scored/conceded features. Match scores use the match
  result score fields from `matches.csv`.
